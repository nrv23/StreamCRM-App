# 02 - Objectives

## Objetivo general

Construir una plataforma CRM modular, escalable y orientada a eventos, usando Node.js, PostgreSQL, Redis, RabbitMQ, Docker y React.

## Objetivos técnicos

### Node.js avanzado

Aplicar:

- Streams.
- Pipeline.
- Backpressure.
- Transform streams.
- Async iterators.
- EventEmitter.
- Worker Threads.
- Child Process.
- AbortController.
- Manejo de memoria en procesos largos.

### PostgreSQL

Aplicar:

- Modelado relacional.
- Constraints.
- Foreign keys.
- Índices compuestos.
- Índices parciales.
- Transacciones.
- Cursores.
- Materialized views.
- Triggers.
- EXPLAIN ANALYZE.
- Paginación eficiente.

### Sistemas distribuidos

Aplicar:

- Event Driven Architecture.
- Producer / Consumer.
- RabbitMQ exchanges.
- Retry queues.
- Dead Letter Queues.
- Saga Pattern.
- Circuit Breaker.
- Consistencia eventual.
- Idempotencia.
- Correlation IDs.

### Redis

Aplicar:

- Cache Aside.
- Sesiones.
- Rate limiting.
- Pub/Sub.
- Locks distribuidos.
- Cache de permisos.
- Cache de métricas.

## Objetivos de producto

El sistema debe permitir:

- Gestionar clientes.
- Importar clientes masivamente.
- Crear campañas.
- Procesar campañas en segundo plano.
- Gestionar tickets.
- Exportar reportes grandes.
- Ver progreso en tiempo real.
- Auditar cambios importantes.
- Notificar eventos relevantes.

## Objetivos de portafolio

El proyecto debe demostrar ante una empresa que el desarrollador puede:

- Diseñar software con criterio.
- Construir APIs mantenibles.
- Trabajar con PostgreSQL más allá de lo básico.
- Usar Node.js para tareas de alto volumen.
- Resolver problemas de memoria usando streams.
- Diseñar workers y colas.
- Implementar resiliencia.
- Documentar arquitectura.
- Pensar como software engineer, no solo como programador de endpoints.
